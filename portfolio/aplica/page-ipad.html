Разработка приложений для iPad

iPad – еще один после iPhone инновационный мобильный продукт от Apple, установивший свои негласные стандарты на рынке планшетных компьютеров, а также нетбуков, прямым конкурентом которых он является.

На iPad, как и на iPhone, устанавливается операционная система iOS, в версии для iPad. Таким образом, разработанное мобильное приложение для iPhone запускается практически без модификаций также и на iPad. Но, как правило, интерфейс приложения для iPhone перерабатывается специально для iPad, чтобы максимально использовать возможности большего экрана.

На что стоит обратить внимание при заказе мобильного приложения для iPad

  1. Все особенности разработки приложения для iPhone применимы и для iPad.
  2. При наличии версии приложения для iPhone версия для iPad может быть выпущена как отдельное приложение только для iPad или в рамках единого приложения для iPhone+iPad. В случае платного приложения пользователь платит за два или за одно приложение соответственно.
  3. iPad существует в двух основных версиях: обычная и 3G. Только версия 3G поддерживает функции геолокации (определения местоположения).
  4. iPad 2 (в отличие от iPad) имеет две видео-камеры, позволяющие создавать приложения с принципиально новыми функциями, например, с аргументированной реальностью.
  5. iPhone может являться для iPad управляющим устройством (через Bluetooth-соединение), что позволяет создавать приложения, работающие на связке нескольких устройств.

Из чего складывается стоимость разработки приложения для iPad?

Если приложение для iPad разрабатывается на основе приложения для iPhone, то, как правило, стоимость приложения для iPad составляет 30-40% от стоимости приложения для iPhone за счет использования практически одного и того же программного кода и того же стиля дизайна интерфейса в случае использования собственного дизайна приложения.

Если приложение разрабатывается только для iPad, то стоимость разработки рассчитывается отдельно по аналогичной для приложения под iPhone схеме:

Этап работ Оценка временных затрат
Техническое задание На основе краткого описания задачи (брифа на разработку)
Дизайн Стандартный Не требуется
Нестандартный На основе количества разрабатываемых эскизов пользовательского интерфейса, указанных в техническом задании
Программирование Стандартный дизайн На основе перечня функций приложения и технических рисков, указанных в техническом задании.
Нестандартный дизайн Затраты на приложение со стандартным дизайном + 50-80%.
Тестирование 5-10% от затрат времени этапа программирования
Публикация приложения в App Store 2-3 часа

В рамках каждого этапа разработки обычно резервируется 5-10% времени на доработки по замечаниям и предложениям.
